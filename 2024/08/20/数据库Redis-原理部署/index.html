redis概述

1.为什么使用redis

  1. 快速的读写速度：Redis是一个基于内存的数据存储系统，因此具有非常快速的读写速度。这使得它非常适合用作缓存系统，可以大大提高应用程序的性能和响应速度。
  2. 支持丰富的数据结构：Redis支持多种数据结构，如字符串、哈希、列表、集合、有序集合等，这使得它可以灵活地处理各种类型的数据，并且支持丰富的操作。这使得Redis不仅仅可以用作缓存，还可以用于实时统计、计数、排行榜等功能。
  3. 高可用性和可扩展性：Redis支持主从复制和集群模式，可以提供高可用性和横向扩展能力。这使得Redis可以应对大规模的数据存储和高并发访问。
  4. 持久化和数据备份：Redis支持将数据持久化到磁盘，这意味着即使发生服务器重启，数据也不会丢失。同时，Redis还支持数据的定期备份和恢复，能够有效地保障数据的安全性和可靠性。
  5. 支持丰富的应用场景：除了作为缓存系统外，Redis还可以用于会话存储、消息队列、实时分析、地理位置查询等多种应用场景。它的灵活性和高性能使得它在各种领域都有广泛的应用。

总之，使用Redis可以提供给用户更好的体验，帮助开发者更好地应对大规模数据和高并发访问的挑战。

无redis场景

对于Web来说，用户量和访问量增一定程度上推动项目技术和架构的更迭和进步。

可能会有以下的一些状况：

页面并发量和访问量并不多，MySQL足以支撑自己逻辑业务的发展。
那么其实可以不加缓存。最多对静态页面进行缓存即可。

页面的并发量显著增多，数据库有些压力，并且有些数据更新频率较低反复被查询或者查询速度较慢。
那么就可以考虑使用缓存技术优化。

对高命中的对象存到key-value形式的Redis中，那么如果数据被命中，那么可以不经过效率很低的db。
从高效的redis中查找到数据。

当然，可能还会遇到其他问题，你还通过静态页面缓存页面、cdn加速、甚至负载均衡这些方法提高系统并发量。

有redis场景

缓存适用于高并发的场景，提升服务容量。

主要是将从经常被访问的数据或者查询成本较高从慢的介质中存到比较快的介质中，比如从硬盘—>内存。

我们知道大多数关系数据库是基于硬盘读写的，其效率和资源有限，而redis是基于内存的，其读写速度差别差别很大。

当并发过高关系数据库性能达到瓶颈时候，就可以策略性将常访问数据放到Redis提高系统吞吐和并发量。

对于常用网站和场景，关系数据库主要可能慢在两个地方：

1.读写IO性能较差
2.一个数据可能通过较大量计算得到

所以使用缓存能够减少磁盘IO次数和关系数据库的计算次数。

读取上速度快也从两个方面体现：

1.基于内存，读写较快
2.使用哈希算法直接定位结果不需要计算，直接key-value获取结果

所以对于像样的，有点规模的网站，缓存是很必要的，而Redis无疑是最好的选择之一。

但是使用缓存，也要注意诸多问题，多引入了一个环节，必然会引发更多问题

  1. 缓存不适用于，数据一致性特别高场景
  2. 缓存也不适用于并发量并不多的，加入缓存，不但架构更难维护，浪费资源，且让项目更臃肿，可能redis也会出问题
  3. 引入缓存，就要考虑对k-v的数据如何设计

数据一致性特别高的为什么不建议redis存储

  1. 数据持久性：尽管Redis提供了持久化功能，但是由于其主要是基于内存的数据库，数据持久化的机制相对于传统的磁盘存储数据库来说可能更容易丧失数据一致性。即使开启了持久化功能，也难以保证和传统数据库一样的数据持久性和稳定性。
  2. 内存限制：Redis的数据存储是基于内存的，因此受到内存容量的限制，如果数据量很大，可能会导致内存不足的问题。而且，将大量数据存储在内存中也会增加成本和复杂性，不利于系统的扩展和维护。
  3. 操作复杂性：在保持数据一致性的情况下，需要进行严格的并发控制和事务管理。虽然Redis提供了一些原子操作和事务支持，但在处理复杂的数据一致性问题时会相对复杂，需要开发人员额外处理和管理。
  4. 数据库更适合的场景：对于数据一致性要求特别高的场景，传统的关系型数据库或者其他分布式数据库可能更适合，这些数据库通常具有更严格的事务支持和数据一致性保证。

2.关系型与非关系型数据库

关系型数据库：mysql、oracle、postgreSQL

非关系：redis mongoDB ES

非关系型数据库:

nosql(not only sql)不仅仅是sql，是不同于传统关系型数据库的统称。

特点是 key-value 这种键值对存储数据的形式，而不是Mysql这样库、表结构存储形式。

noSQL用于处理高速读写的数据存储场景

如用户访问行为，用户地理定位数据、电商秒杀类的订单数据超高的 瞬时并发请求

视频网的弹幕信息、直播间的刷礼物、点赞等功能

联盟、王者的玩家巅峰排行榜的数据

微博的最新评论、最新点赞

这类要求高并发，高读写的业务场景，数据都是存储在redis里，提高APP响应速度，让用户更好的体验

3.redis特点

Redis是一种支持key-value等多种数据结构的存储数据库系统

特点 说明
1.软件源码高质量，运行速度快，C语言开发，单线程架构 质量高：软件源码经过测试与审计才上线
运行速度快：C编译性语言开发的，编译后生成机器码运行，速度快
单线程架构：处理数据更简单，一个个数据处理速度更快
总之就是redis这个应用程序处理起来数据很快
2.支持丰富数据类型，面对各种业务场景 字符串、哈希、列表、集合、有序集合
3.功能丰富，可以实现如 计数器（点赞评论数）、过期功能（购物车订单）、 消息队列、发布订阅
4.支持多种语言客户端连接 php，java，golang，python
5.支持数据持久化 运行时的数据都在内存，可以基于RDB，AOF持久化到磁盘里，或混合持久化
6.内置高可用架构 主从复制，哨兵，集群
  1. 内存数据结构存储:
    • Redis将数据完全存储在内存中,提供了丰富的数据结构,如字符串、哈希、列表、集合、有序集合等。
    • 这使得Redis能够提供非常高的读写性能。
  2. 持久化:
    • Redis支持将内存中的数据异步或同步地保存到磁盘上,以实现数据的持久化。
    • 主要持久化方式包括RDB(Redis数据库备份)和AOF(附录文件)。
  3. 复制(主从复制):
    • Redis支持主从复制,可以实现一个主Redis服务器向多个从Redis服务器同步数据。
    • 这提高了读取性能,并增加了数据的可用性和容错性。
  4. 事务:
    • Redis支持将一组命令作为一个原子性的事务执行。
    • 事务中的所有命令要么全部执行,要么全部不执行。
  5. 发布/订阅模式:
    • Redis提供了发布/订阅的消息模式,支持订阅者订阅感兴趣的频道,并接收发布者发布的消息。
  6. Keys过期机制:
    • Redis支持为键设置过期时间,到期后自动删除。
    • 这对于缓存应用很有用。
  7. 高可用和分布式:
    • Redis支持主从复制和哨兵模式,提供了一定程度的高可用性。
    • 并且通过Redis集群,可以实现水平扩展和分布式存储。

总之,Redis凭借其高性能、丰富的数据结构、持久化和复制机制,广泛应用于缓存、消息队列、排行榜等场景。它是一个非常灵活和强大的NoSQL数据库。

redis特点通俗理解

  1. 多种数据结构:
    • 就像仓库里有不同的箱子，Redis有各种“箱子”来存放不同类型的数据，比如可以放文本、数字列表、集合等。
  2. 超快速度:
    • Redis就像一个超级快的快递员，可以迅速地存取和检索信息，几乎瞬间就能完成任务。
  3. 数据自动保存:
    • 就像你把重要的笔记拍照保存，Redis也会在不小心丢失信息时帮你备份，保证数据的安全。
  4. 复制数据:
    • 就像给重要的文件做副本以防丢失，Redis可以创建数据的副本，这样即使一个服务器出问题，数据也不会丢失。
  5. 事务操作:
    • 在Redis里执行一系列操作就像订一张票，要么所有步骤都完成，要么都不完成，确保不会出现半途而废的情况。
  6. 发布订阅:
    • 类似于订阅杂志，你可以在Redis上订阅特定的“频道”，一旦有消息发布到这个频道，你就能立即知道。
  7. 自动清理过期数据:
    • 就像超市里的过期食品会自动下架，Redis中的数据如果超过了设定的“保质期”就会自动被清理。
  8. 高可用性和分布式存储:
    • 这就像是备份你的文件到云端，确保即使家里的电脑坏了，你的文件也能在其他地方安全访问。Redis可以通过多个服务器来确保数据的可用性和扩展性。

4.redis应用场景-核心

Redis作为一种高性能的内存数据结构存储系统，应用场景非常广泛。以下是一些常见的使用Redis的场景：

  1. 高速缓存:
    • 使用Redis作为数据库或应用服务器的缓存层，可以快速检索数据，减少数据库访问压力，提高应用响应速度。
  2. 会话管理:
    • 存储用户的登录状态和会话信息，例如用户的登录时间、权限信息等。
  3. 排行榜/计数器:
    • 实现实时排行榜功能，比如游戏评分、新闻热度等，也可以用于计数器，如统计网站访问量。
  4. 消息队列:
    • 作为消息中间件，用于处理高并发的消息传递，实现应用程序之间的解耦合。
  5. 社交网络:
    • 用于存储用户关系（如好友列表）、用户偏好设置、内容推荐等。
  6. 地理信息空间:
    • 存储地理位置数据，进行地理查询和空间分析。
  7. 图像/视频处理:
    • 缓存图片和视频内容，减少对原始存储系统的访问。
  8. 电子商务:
    • 缓存商品信息、用户购物车、订单状态等，提高网站性能。
  9. 游戏开发:
    • 用于存储游戏状态、玩家数据、排行榜等。
  10. 实时分析:
    • 对实时数据进行分析和处理，如股票交易数据、日志分析等。
  11. 分布式系统:
    • 用于分布式锁、分布式计数器等分布式系统的组件。
  12. 物联网 (IoT):
    • 用于快速处理来自物联网设备的实时数据。

Redis的这些应用场景得益于其高性能、易用性和丰富的数据结构支持，使其成为现代软件架构中不可或缺的一部分

通俗理解一些应用场景

  1. 高速缓存：
    • 想象你经常去一个很大的书店，每次找书都要翻很久。你聪明地决定把常读的书放在家里的书架上。Redis就像你家的书架，你经常访问的网页内容或数据都存放在这里，下次访问时就不用再到书店（数据库）去找了，直接在家（Redis）就能快速拿到，节省了很多时间。
  2. 会话管理：
    • 上了网就像进了商场，每次想要买的东西和购物篮都需要记录。Redis就像商场里的一个小本本，记着你进了哪些店，买了什么，下次再来时，可以直接从本子上找到，不用再到处找服务员问。
  3. 排行榜/计数器：
    • 想象你参加了一个比赛，想知道自己在所有参赛者中的排名。Redis在这里就像的成绩公布板，每次有人参赛，板子上的数据就会更新，你的排名一目了然。
  4. 消息队列：
    • 就像你在餐厅里用点餐夹子把你的订单夹在队伍里，Redis帮你管理好这个订单队列。无论有多少人同时点餐，你的订单都能按顺序得到处理。
  5. 社交网络：
    • 在社交网络里，你想知道谁关注了你，或者你关注了谁。Redis就像一个快速查找表，帮你快速找到所有关注的人和被关注的人。
  6. 地理信息空间：
    • 想要找附近的餐厅或者加油站，你可以使用Redis来存储地图上的各种地点，快速告诉你附近有什么。
  7. 图像/视频处理：
    • 在网上看视频或者图片，这些内容都很大，加载起来很慢。Redis就像一个快递员，把常用的图片和视频提前送到你的电脑上，让你能快速打开。
  8. 电子商务：
    • 在网上购物，你想看看别人买了什么，或者想要的东西库存还有多少。Redis帮你快速查到这些信息，让你购物体验更好。
  9. 游戏开发：
    • 玩游戏时，你的分数、道具和角色状态都需要实时保存。Redis就像一个快速存取的小盒子，帮你保存这些信息。
  10. 实时分析：
    • 想知道现在有多少人在看一场直播，或者有多少人打开了你的网站。Redis就像一个快速计算器，帮你实时统计这些数据。

这些场景展示了Redis如何通过其快速的数据存储和检索能力，在各个行业中提升效率和提高用户体验。

知识点补充：

MALLOC=libc参数说明：

MALLOC=libc的作用是告诉编译器在编译软件时使用 libc 提供的内存分配函数来管理内存，而不是系统默认的内存分配函数。这可以影响软件的性能和可移植性，有时候可以在特定环境下提供更好的性能或确保代码在不同系统上的可移植性。

tcl与tcl-devel说明：

tcl和tcl-devel是与 Tcl（Tool Command Language）编程语言相关的软件包。它们的作用如下：

  1. tcl：这是 Tcl 编程语言的运行时库，它包含了执行 Tcl 脚本所需的运行时环境。使用 tcl 包，可以在系统上运行 Tcl 脚本和程序。
  2. tcl-devel：这是 Tcl 编程语言的开发包，包含了用于开发 Tcl 程序的头文件和静态库文件，以及其他开发所需的工具和文档。使用 tcl-devel包，可以在系统上开发和编译使用 Tcl 的程序。

总而言之，tcl 和 tcl-devel提供了运行时环境和开发工具，让用户能够在系统上运行和开发使用 Tcl 编程语言的程序。

redis管理命令说明：

redis-benchmark 是Redis自带的基准测试工具，用于测试Redis服务器的性能。它可以模拟多个并发客户端对 Redis 服务器进行读写操作，以评估服务器的性能。通过执行 redis-benchmark 命令，可以测试 Redis 服务器在当前环境下的读取和写入性能，并获得各种指标数据，如每秒操作数量、平均响应时间等，以便进行性能分析和优化。

redis-check-aof 是 Redis 提供的用于检查和修复 AOF（Append-Only File）文件的工具。AOF 是 Redis 使用的一种持久化方式，它记录了所有写命令，以便在服务器重启时重新执行这些命令来还原数据。redis-check-aof 工具可以用于检查 AOF 文件的完整性，查找可能出现的错误，并尝试修复这些错误。这有助于确保 AOF 文件的可靠性和一致性，从而保护数据不受损坏。

redis-check-rdb 是 Redis 提供的用于检查和修复 RDB（Redis DataBase）文件的工具。RDB 文件是 Redis 使用的另一种持久化方式，它是一个二进制文件，用于将 Redis 数据集保存到磁盘上。redis-check-rdb 工具可以用于检查 RDB 文件的完整性，查找可能出现的错误，并尝试修复这些错误。这有助于确保 RDB 文件的可靠性和一致性，从而保护数据不受损坏。
