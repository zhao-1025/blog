3.危险命令禁止操作

线上生产环境禁止执行下方命令

错误操作

下面这个操作是在1200w的key中模糊查找相关的value信息，是严重错误的操作

在1200w个key中搜索时，redis会调用大量硬件资源，当硬件资源不足会导致redis有挂掉的风险

redis是单线程软件，执行keys命令相对redis这个服务来说是很慢的，会影响其他请求进行等待造成阻塞的状态

为什么不能使用keys * 相关命令

在生产环境下不建议使用 keys * 命令，因为它会返回所有匹配的 key，这可能会导致性能问题。当 Redis 数据库中包含大量 key 时，执行 keys * 命令会导致 Redis 服务器在查找和返回所有 key 时消耗大量的 CPU 和内存资源，在服务器中不止redis一个软件，还有多个服务程序在运行占用硬件资源，而出现硬件资源抢夺时，会导致 Redis 服务器性能下降甚至崩溃。

另外，keys * 命令是发生命令阻塞的，redis是单线程的软件，每次只执行一次响应与命令，一旦在同时有其他 Redis 命令执行会被阻塞无法执行，因为执行keys*会是一个相对漫长的过程，直到 keys * 命令执行完毕。这会对其他客户端请求造成延迟。

如果需要获取所有 key，推荐使用 SCAN 命令来迭代遍历所有 key，这样可以避免阻塞 Redis 服务器，并且可以逐步获取 key 而不会一次性返回所有 key。

官网说明：https://redis.io/docs/manual/data-types/

优质文章：https://xiaolincoding.com/redis/base/redis_interview.html

首先对redis来说，所有的key（键）都是字符串。

我们在谈基础数据结构时，讨论的是存储值的数据类型

主要包括常见的5种数据类型，分别是：String、List、Set、Zset、Hash。

结构类型 结构存储的值 结构的读写能力
String字符串 可以是字符串、整数或浮点数 对整个字符串或字符串的一部分进行操作；对整数或浮点数进行自增或自减操作；
List列表 一个链表，链表上的每个节点都包含一个字符串 对链表的两端进行push和pop操作，读取单个或多个元素；根据值查找或删除元素；
Set集合 包含字符串的无序集合 字符串的集合，包含基础的方法有看是否存在添加、获取、删除；还包含计算交集、并集、差集等
Hash散列 包含键值对的无序散列表 包含方法有添加、获取、删除单个元素
Zset有序集合 和散列一样，用于存储键值对 字符串成员与浮点数分数之间的有序映射；元素的排列顺序由分数的大小决定；包含方法有添加、获取、删除单个元素以及根据分值范围或成员来获取元素

1.字符串string类型

字符串类型是最简单和最常见的数据类型之一

用户可以使用字符串类型来存储任意类型的数据，包括文本、数字或二进制数据

下面是一些关于Redis字符串类型的基本信息：

  1. 存储：Redis字符串类型可以存储最大512MB的数据。
  2. 操作：用户可以对字符串类型进行常见的操作，如设置值、获取值、追加值、覆盖值等。
  3. 命令：用户可以使用一系列的命令来操作字符串类型，如SET、GET、APPEND等。
  4. 应用：字符串类型广泛应用于缓存、计数器、会话管理等各种场景。

下面是一些常见的Redis字符串类型的命令：

  • SET key value：设置指定键的值为指定的字符串。
  • GET key：获取指定键的值。
  • APPEND key value：将指定值追加到指定键的值的末尾。
  • INCR key：对存储在指定键的值递增1。
  • DECR key：对存储在指定键的值递减1。

1.4应用场景

  1. 缓存数据：企业可以使用Redis字符串类型来缓存经常访问的数据，例如产品信息、用户信息和其他常用数据。这有助于提高应用程序的性能和降低数据库负载。
  2. 会话管理：Redis字符串类型可用于存储用户会话信息和身份验证令牌。通过将会话数据存储在Redis中，企业可以实现分布式会话管理，确保用户在多个应用服务器之间的无缝登录状态。
  3. 缓存限制：企业可以使用Redis字符串类型来实现访问频率限制和缓存限制。通过在Redis中存储请求次数和过期时间等信息，可以有效地控制对重要资源的访问。
  4. 计数器：Redis字符串类型可用于实现计数器功能，用于跟踪用户活跃度、产品销量等指标。这对于监控和分析数据在企业中非常有用。
  5. 消息队列：Redis的字符串类型可用作简单的消息队列，用于在不同系统之间传递消息和任务。这对于实现异步通信和数据处理非常有用。
  6. 临时状态存储：企业可以使用Redis字符串类型来存储临时状态数据，如临时锁定、临时会话状态等。这有助于在应用程序中保存临时性数据。

2.列表list类型

Redis的List类型是一个双向链表结构，它可以存储多个字符串元素，并且允许在两端进行快速的插入和删除操作。List类型在企业中有许多应用场景，以下是一些通俗易懂的例子：

  1. 消息队列：类似于排队等待接待的顾客，企业可以使用Redis的List类型来实现简单的消息队列。当有新的消息需要处理时，可以将消息放入List的尾部，而消费者则可以从List的头部获取消息进行处理，实现简单的消息队列功能。
  2. 实时数据流：企业需要实时监控和处理数据，比如实时日志记录、实时事件处理等。Redis的List类型可以用来存储实时生成的数据，以便后续处理和分析。
  3. 最新消息推送：类似于社交网络中的动态消息推送，企业可以使用List类型来存储用户的最新消息，新消息在List的头部插入，用户则可以从头部获取最新的消息。
  4. 任务队列：企业中经常需要处理一些异步任务，比如后台数据处理、邮件发送等。Redis的List类型可以用来存储待处理的任务，而工作者则可以从List中获取任务进行处理，实现简单的任务队列功能。

通俗理解

当我们谈到 Redis 中的 List 类型时，最好的比喻就是它是一个类似于数组的数据结构，可以存储一系列有序的元素。这些元素可以按照插入的顺序进行访问。

下面是一些通俗易懂的 List 类型在 Redis 中的应用场景：

  1. 微信公众号的粉丝列表：将每个关注公众号的用户ID添加到 List 中，这样就可以方便地获取最近关注公众号的用户。类似于有一个按照时间顺序排列的粉丝名单。

  3. 任务队列：一个常见的场景就是将需要处理的任务以列表的形式存储在 Redis 中，比如处理用户上传的图片、视频等任务。
  4. 商品订单列表：将用户购买的商品订单按照时间顺序存储在 List 中，可以方便地获取用户最近的订单记录。

3.集合set类型

在Redis中，集合（Set）是一种无序且不重复的数据结构，它类似于数学中的集合概念。Redis的集合类型提供了一系列的操作，允许你对集合进行添加、删除和查找元素，还支持集合间的交集、并集、差集等操作。

Redis 集合类型的特点和用途：

  1. 无序性：集合中的元素是无序的，即元素的存储顺序与其添加顺序无关。
  2. 唯一性：集合中不允许重复的元素，每个元素在集合中只能出现一次。
  3. 快速的添加、删除和查找操作：Redis提供了高效的添加（SADD）、删除（SREM）和成员查找（SISMEMBER）操作，时间复杂度通常为 O(1)。不管set集合vaule中的元素有多少增删改查的速度都是一样，不会影响元素多会造成处理速度慢的情况。O(1)是算法的时间复杂度的一个标识。
  4. 集合操作：Redis支持对集合进行交集（SINTER）、并集（SUNION）、差集（SDIFF）等操作，这些操作可以用来处理多个集合的数据。
  5. 适合于标签和关系型数据建模：集合在标签（tags）系统、社交网络中用户的关注列表、共同好友等场景中非常有用，能够方便地管理和操作数据的交集和并集关系。

4.散列hash类型

在Redis中，哈希（Hash）是一种用于存储键值对集合的数据类型。哈希结构在Redis中类似于一个字典（Dictionary）或映射（Map），它可以存储多个字段（field）及其对应的值（value）。哈希特别适合用于表示对象及其属性，例如用户信息、商品信息等。存储结构化的数据，使得数据的访问和操作更加便捷高效。

4.1命令说明

  1. 设置字段值

    • HSET key field value：在指定的哈希中设置字段的值。

    示例：HSET user:1000 name "Alice"

  2. 获取字段值

    • HGET key field：获取指定哈希中字段的值。

    示例：HGET user:1000 name

  3. 删除字段

    • HDEL key field1 [field2 ...]：从指定的哈希中删除一个或多个字段。

    示例：HDEL user:1000 email

  4. 检查字段是否存在

    • HEXISTS key field：检查指定的哈希中是否存在指定的字段。

    示例：HEXISTS user:1000 age

  5. 获取所有字段和值

    • HGETALL key：获取指定哈希中所有字段和对应的值。

    示例：HGETALL user:1000

  6. 获取所有字段名

    • HKEYS key：获取指定哈希中所有字段的名字。

    示例：HKEYS user:1000

  7. 获取所有字段值

    • HVALS key：获取指定哈希中所有字段的值。

    示例：HVALS user:1000

  8. 获取字段数量

    • HLEN key：获取指定哈希中字段的数量。

    示例：HLEN user:1000

  9. 设置多个字段值

    • HMSET key field1 value1 [field2 value2 ...]：在指定的哈希中设置多个字段的值。

    示例：HMSET user:1000 name "Alice" age 30 email "alice@example.com"

  10. 获取多个字段值

    • HMGET key field1 [field2 ...]：从指定的哈希中获取多个字段的值。

    示例：HMGET user:1000 name age email

4.7应用场景

Redis 哈希类型在生产环境中的应用非常广泛，以下是对这些应用场景的解释：

  1. 用户信息存储：
    • 将用户的个人信息（如用户名、密码、邮箱、手机号码）存储在哈希类型中。
    • 这样做的好处是可以快速读取和更新这些信息，非常适合需要频繁访问用户数据的应用场景。
  2. 商品信息存储：
    • 在电商平台上，可以将每个商品的详细信息（如名称、价格、库存、描述）存储在哈希类型中。
    • 这样可以快速查询和更新商品信息，确保用户在购物时能够获得最新的商品状态。
  3. 会员卡信息存储：
    • 将会员卡的信息（如会员卡号、会员等级、积分）存储在哈希类型中。
    • 这种方式能够快速访问和更新会员信息，提升用户体验，特别是对于积分的实时更新。
  4. 实时监控数据存储：
    • 监控系统中使用哈希类型存储服务器的各项性能指标（如CPU使用率、内存使用情况、网络流量）。
    • 这种方式可以实时监控系统状态，及时发现和解决问题，保证系统的正常运行。
  5. 地理位置信息存储：
    • 对于位置服务应用，可以将地理位置信息（如经纬度、地址名称、附近商家信息）存储在哈希类型中。
    • 这样可以实现快速的地理位置查询和附近商家推荐，提升用户体验。
  6. 任务调度信息存储：
    • 在任务调度系统中，使用哈希类型存储每个任务的相关信息（如任务名称、执行时间、状态）。
    • 这种方式能够方便地调度任务和监控任务状态，提高系统的灵活性和可靠性。

Redis哈希类型最多的应用场景

用户信息存储和商品信息存储是Redis哈希类型最常见的应用场景。这是因为这些场景需要频繁地更新和快速地访问数据。Redis哈希类型能够高效地利用内存，并提供快速的读写操作，非常适合这种结构化数据的存储需求。

通过使用Redis哈希类型，开发者可以显著提升应用的性能和响应速度，提供更好的用户体验。

5.有序集合zset类型

Redis 有序集合和集合一样也是 string 类型元素的集合,且不允许有重复元素，唯一区别就是元素与分数关联

Redis 的有序集合（ZSet）是一种特殊的数据类型，它将元素与分数（score）关联，每个元素是唯一的，分数则用于对元素进行排序。这个结构使得 ZSet 既能像集合一样存储独特的元素，又能通过分数对元素进行排序，非常适合处理需要排序或优先级处理的应用场景。

ZRANGE 根据元素在有序集合中所处的位置，从有序集合中获取多个元素 ZRANGE zset-key 0-1 withccores
ZREM 如果给定元素成员存在于有序集合中，那么就移除这个元素 ZREM zset-key member1

应用场景

Redis 的有序集合（ZSet）数据类型在生产环境中的应用场景也非常广泛，主要包括以下几类：

  1. 排行榜系统：
    • 例如游戏中的积分排行榜、网站中的积分排行榜等。
    • 使用 ZSet 可以快速地插入、删除和更新用户积分，同时能够根据积分进行排序，方便取出前 N 名用户。
  2. 延迟队列：
    • 任务队列系统中，用于存储需要延迟执行的任务。
    • 通过设置任务的执行时间为 score，ZSet 能够按时间排序并在任务到期时进行处理。
  3. 缓存热点数据：
    • 存储经常访问的数据或热门内容，例如热门文章、热门商品等。
    • ZSet 根据访问次数或权重进行排序，方便快速获取热门数据。
  4. 带权重的消息系统：
    • 存储带有优先级的消息，例如客户服务系统中的工单处理。
    • ZSet 根据消息的权重进行排序，优先处理权重高的消息。
  5. 分数排名统计：
    • 在投票系统中，统计选项的得票数。
    • ZSet 能够根据得票数进行排序，方便展示得票情况。
  6. 存储带有优先级的任务：
    • 在调度系统中，存储需要按优先级执行的任务。
    • 通过 ZSet 的排序特性，能够优先执行高优先级任务。

简单通俗理解

Redis 的有序集合（ZSet）数据类型在生产环境中的应用非常广泛，以下是对这些应用场景的简单通俗解释：

  1. 排行榜系统：
    • 例如在游戏中，可以将玩家的积分存储在 ZSet 中，通过积分排序生成排行榜。
    • 这样做的好处是能够快速更新和获取排行榜信息，保证玩家排名的实时性。
  2. 延迟队列：
    • 比如在任务调度系统中，有些任务需要在未来某个时间点执行，可以将这些任务和它们的执行时间存储在 ZSet 中。
    • 通过设置任务的执行时间为 score，可以按时间排序并在任务到期时执行。
  3. 缓存热点数据：
    • 存储经常访问的数据或热门内容，例如热门文章、热门商品等。
    • 根据访问次数或权重排序，快速获取热门数据，提升用户体验。
  4. 带权重的消息系统：
    • 存储带有优先级的消息，例如客户服务系统中的工单处理。
    • 根据消息的优先级进行排序，确保优先处理重要消息。
  5. 分数排名统计：
    • 在投票系统中，统计各选项的得票数，通过 ZSet 进行排序。
    • 方便展示投票结果和排名。
  6. 存储带有优先级的任务：
    • 在任务调度系统中，存储需要按优先级执行的任务。
    • 通过 ZSet 的排序特性，优先执行高优先级任务，保证关键任务的及时处理。

Redis 有序集合（ZSet）类型最多的应用场景

排行榜系统和延迟队列是 ZSet 类型最常见的应用场景。这是因为这些场景需要对数据进行排序和优先级处理，而 ZSet 能够高效地实现这些需求，确保数据的实时性和系统的高效运行。
